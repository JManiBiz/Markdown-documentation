Introduction

Markdown is a lightweight markup language that is used to add formatting elements to plaintext documents. In 2004, John Gruber and Aaron Swartz came together and created Markdown with the sole goal of it being readable even in code form. This allows it to be one of the world's most popular markup languages.

Formatting with Markdown is different from using a WYSIWYG editor like Microsoft Word. In Word, formatting, such as bold and italics, is done with the click of a button and the change is viewed instantly. In Markdown, the formatting is done with the keys on the keyboard allowing for a fluid motion while typing. Most of the time, the changes can not be viewed instantly but can be seen in a web browser. However, many applications do allow you to preview the formatted document in real-time. The joy of Markdown syntax is that it can still be easily read without it being rendered.

There are many reasons why Markdown is used. Markdown is a very versatile markup language that can be used for everything. Websites, documents, notes, etc can all be made with Markdown. It is portable because Markdown files can be opened using any text editor or Markdown specific editors. Because of this, Markdown is also platform-independent and can be created on any device with any operating system.

Syntax

The syntax for Markdown is easy to pick up and there isn't a steep learning curve. It can be catergorized into two (2) things: Basic Syntax and Extended Syntax. Almost all Markdown processors support the basic syntax that was created by John Gruber and Aaron Swartz. While the basic syntax is enough for ordinary, day-to-day use, the extended syntax adds for functionality to Markdown. It brings with it the ability to add tables, footnotes, fenced code blocks and automatic URL linking. Not all Markdown processors support extended syntax without the use of extensions.

Headings

Pound signs (#) are added in front of a word or phrase to create a heading. The number of pound signs used should correspond to the heading level. For example, a heading level 4 (<h4>) is created by using four pound signs (e.g., #### Header).

# Heading 1

Heading 1
## Heading 2

Heading 2
### Heading 3

Heading 3
#### Heading 4

Heading 4
##### Heading 5
Heading 5
###### Heading 6
Heading 6

Emphasis

Bold and italics can be used to add emphasis to text

Bold

To bold text, two asterisks or underscores are added before and after a word or phrase. To bold the middle of a word for emphasis, add two asterisks without spaces around the letters.

Markdown Syntax Rendered Output
This is **important.**
This is __important.__

This is important.
Hope**is**strong.

Hopeisstrong.
Italic

To italicize text, one asterisk or underscore is added before and after a word or phrase. To italicize the middle of a word for emphasis, add one asterisk without spaces around the letters.

Markdown Syntax Rendered Output
Dog- *Canis lupus familiaris*
Dog- _Canis lupus familiaris_

Dog- Canis lupus familiaris
Bold and Italic

For extra emphasis, text can be bold and italicized at the same time. To do this, three asterisks or underscores before and after a word or phrase. To bold and italicize the middle of a word for emphasis, add three asterisks without spaces around the letters.

Markdown Syntax Rendered Output
This is ***really very important.***
This is ___really very important.___
This is __*really very important.__*
This is **_really very important.**_ 		This is really very important.
This is some really***very***important text. This is some reallyveryimportant text.
Blockquotes

Blockquotes are used to quote text from another source. To create a blockquote, add a > in front of a paragraph.

> You could leave life right now. Let that determine what you do and say and think. - Marcus Aurelius

The rendered output would like:

You could leave life right now. Let that determine what you do and say and think. - Marcus Aurelius
Blockquotes with Multiple Paragraphs

To create blockquotes with multiple paragraphs, add a > on the blank lines between the paragraphs.

>Live a good life. If there are gods and they are just, then they will not care how devout you have been, but will welcome you based on the virtues you have lived by.
>
>If there are gods, but unjust, then you should not want to worship them. If there are no gods, then you will be gone, but will have lived a noble life that will live on in the memories of your loved ones.

The rendered output would be:

Live a good life. If there are gods and they are just, then they will not care how devout you have been, but will welcome you based on the virtues you have lived by.

If there are gods, but unjust, then you should not want to worship them. If there are no gods, then you will be gone, but will have lived a noble life that will live on in the memories of your loved ones.

Nested Blockquotes

Blockquotes can be nested. Add a >> in front of the paragraph to be nested.

>Live a good life. If there are gods and they are just, then they will not care how devout you have been, but will welcome you based on the virtues you have lived by.
>
>>If there are gods, but unjust, then you should not want to worship them. If there are no gods, then you will be gone, but will have lived a noble life that will live on in the memories of your loved ones.

The rendered output would be:

Live a good life. If there are gods and they are just, then they will not care how devout you have been, but will welcome you based on the virtues you have lived by.

If there are gods, but unjust, then you should not want to worship them. If there are no gods, then you will be gone, but will have lived a noble life that will live on in the memories of your loved ones.
